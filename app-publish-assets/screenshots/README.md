# App Screenshots Guide

## Overview

High-quality screenshots are critical for app store conversion rates. Users make decisions based on visual presentation, so screenshots must be professional, clear, and showcase key features.

## Screenshot Strategy

### Primary Goals
1. Show main features visually
2. Demonstrate user interface quality
3. Highlight unique value propositions
4. Build trust with professional presentation
5. Optimize for conversion (downloads)

### Key Features to Showcase
1. **AI-Powered Analysis** - Light curve upload and detection results
2. **Exoplanet Database** - Browse interface with thousands of planets
3. **3D Visualization** - Interactive planetary system view
4. **Discovery Dashboard** - Personal statistics and achievements
5. **Learning Center** - Educational resources and tutorials
6. **Planet Details** - Comprehensive information cards

## Google Play Store Requirements

### Phone Screenshots (Required)
- **Quantity:** Minimum 2, maximum 8 (recommend 6-8 for best presentation)
- **Dimensions:**
  - Minimum: 320px on short side
  - Maximum: 3840px on long side
  - Aspect ratio: Between 16:9 and 9:16
- **Recommended sizes:**
  - Portrait: 1080 x 1920 pixels (most common)
  - Portrait: 1440 x 2560 pixels (QHD)
  - Landscape: 1920 x 1080 pixels (if app supports landscape)
- **Format:** PNG or JPG (24-bit RGB, no alpha)
- **File size:** Max 8MB per screenshot
- **Order:** Screenshots display in upload order (first is most important)

### 7-inch Tablet Screenshots (Optional but Recommended)
- **Quantity:** Minimum 2, maximum 8
- **Dimensions:**
  - Portrait: 1200 x 1920 pixels
  - Landscape: 1920 x 1200 pixels
- **Format:** PNG or JPG
- **File size:** Max 8MB per screenshot

### 10-inch Tablet Screenshots (Optional but Recommended)
- **Quantity:** Minimum 2, maximum 8
- **Dimensions:**
  - Portrait: 1600 x 2560 pixels
  - Landscape: 2560 x 1600 pixels
- **Format:** PNG or JPG
- **File size:** Max 8MB per screenshot

### Wear OS Screenshots (Not Applicable)
ExoHunter AI does not support Wear OS.

## Apple App Store Requirements

### iPhone Screenshots (Required)
Must provide screenshots for at least one of each device size class.

#### iPhone 6.9" Display (Required - iPhone 16 Pro Max)
- **Dimensions:** 1320 x 2868 pixels (portrait) or 2868 x 1320 pixels (landscape)
- **Quantity:** Minimum 3, maximum 10
- **Format:** PNG or JPG

#### iPhone 6.7" Display (Required - iPhone 15 Pro Max, 14 Pro Max)
- **Dimensions:** 1290 x 2796 pixels (portrait) or 2796 x 1290 pixels (landscape)
- **Quantity:** Minimum 3, maximum 10
- **Format:** PNG or JPG

#### iPhone 6.5" Display (Required - iPhone 11 Pro Max, XS Max)
- **Dimensions:** 1242 x 2688 pixels (portrait) or 2688 x 1242 pixels (landscape)
- **Quantity:** Minimum 3, maximum 10
- **Format:** PNG or JPG

#### iPhone 5.5" Display (Optional - iPhone 8 Plus, 7 Plus)
- **Dimensions:** 1242 x 2208 pixels (portrait) or 2208 x 1242 pixels (landscape)
- **Quantity:** Minimum 3, maximum 10
- **Format:** PNG or JPG

### iPad Screenshots (Recommended)

#### iPad Pro (6th Gen) 12.9" Display
- **Dimensions:** 2048 x 2732 pixels (portrait) or 2732 x 2048 pixels (landscape)
- **Quantity:** Minimum 3, maximum 10
- **Format:** PNG or JPG

#### iPad Pro (2nd Gen) 12.9" Display
- **Dimensions:** 2048 x 2732 pixels (portrait) or 2732 x 2048 pixels (landscape)
- **Quantity:** Minimum 3, maximum 10
- **Format:** PNG or JPG

### General iOS Requirements
- **Color space:** sRGB or P3
- **File size:** No official limit but keep under 5MB for performance
- **No alpha channels:** Solid backgrounds required
- **No rounded corners:** iOS applies corners automatically

## Screenshot Content Guidelines

### What to Include
- Actual app interface (not mockups)
- High-quality, clear imagery
- Representative of actual user experience
- Key features and functionality
- Professional presentation
- Localized content if supporting multiple languages

### What to Avoid
- Placeholder content or "Lorem ipsum"
- Outdated screenshots that don't match current version
- Low resolution or blurry images
- Screenshots from other apps
- Misleading content not in actual app
- Device frames that cover content (unless using official templates)
- Excessive text overlays that obscure UI
- Personal or sensitive user data

### App Store Compliance
- **No superlatives:** Don't use "best", "#1", "top", "leading"
- **Factual only:** Show actual features, not aspirational concepts
- **No fake reviews:** Don't include testimonials or ratings
- **No competitor references:** Don't mention other apps
- **Accurate representation:** Everything shown must exist in app

## Recommended Screenshot Sequence

### Sequence for Phone/iPhone (6-8 screenshots)

1. **Hero/Main Feature** - AI Analysis Interface
   - Show light curve upload interface with analysis button
   - OR show analysis results with detected exoplanet

## Creating Screenshots

### Method 1: Physical Device Capture (Recommended)
**Android:**
1. Connect Android device to computer
2. Enable USB debugging
3. Run app on device
4. Capture: Power + Volume Down buttons
5. Transfer screenshots from device
6. Use exact screen dimensions (no scaling)

**iOS:**
1. Run app on physical iPhone/iPad
2. Navigate to feature-rich screens
3. Capture: Side button + Volume Up (newer models)
4. Transfer screenshots via AirDrop or cable
5. Screenshots auto-save at correct resolution

### Method 2: Emulator/Simulator
**Android Emulator:**
1. Open Android Studio
2. Create AVD with target screen size (e.g., Pixel 5 - 1080x2340)
3. Launch emulator
4. Run app
5. Use emulator's screenshot tool (camera icon)
6. Save to screenshots folder

**iOS Simulator:**
1. Open Xcode
2. Select target device (iPhone 16 Pro Max, iPad Pro, etc.)
3. Run app in simulator
4. Command + S to save screenshot
5. Choose save location
6. Simulator captures at exact device resolution

### Method 3: Browser DevTools (Web Version)
1. Open app in browser (Chrome/Firefox)
2. Open DevTools (F12)
3. Enable device mode
4. Select device preset (iPhone 14 Pro, Pixel 5, etc.)
5. Set exact dimensions if needed
6. Capture: DevTools screenshot tool or browser extension
7. Ensure high DPI/retina display mode

### Method 4: Screenshot Automation Tools
- **Fastlane Snapshot** (iOS) - Automated screenshots
- **Screengrab** (Android) - Automated screenshots
- **Appetize.io** - Cloud-based device screenshots
- **BrowserStack** - Real device screenshots

### Text Overlay Guidelines (Optional)
- **Font:** Clean, sans-serif (Roboto, SF Pro, Inter)
- **Size:** Large enough to read in thumbnails
- **Position:** Top or bottom, consistent across screenshots
- **Background:** Semi-transparent overlay for readability
- **Color:** High contrast with background
- **Content:** Brief feature description (4-8 words)
- **Alignment:** Centered or left-aligned consistently

## Quality Standards

### Technical Quality
- **Resolution:** Native device resolution (no upscaling)
- **Clarity:** Sharp, no blur or compression artifacts
- **Color:** Accurate colors, consistent across screenshots
- **Alignment:** Straight, no tilted captures
- **Cropping:** Precise, no partial UI elements at edges
- **File format:** PNG for UI with sharp edges, JPG for photos
- **File size:** Optimized (<2MB per screenshot)

### Content Quality
- **Realistic data:** Use real or realistic content (no placeholder text)
- **Complete UI:** All UI elements loaded and visible
- **No errors:** No loading spinners, error messages, or empty states
- **Professional:** Clean, organized, polished appearance
- **Consistent:** Same theme/mode across all screenshots
- **Localized:** Match app's default language (English)

### User Experience
- **Feature clarity:** Each screenshot clearly shows a specific feature
- **Visual appeal:** Attractive, modern design visible
- **Value proposition:** Benefits obvious from visuals
- **Call to action:** Encourages download/exploration
- **Story flow:** Screenshots tell a coherent story when viewed in sequence

## Tips for Success

### Best Practices
- Capture in light mode (generally more appealing) unless dark mode is signature feature
- Show screens with data, not empty states
- Use realistic but professional-looking sample data
- Capture during optimal app state (loaded, no errors)
- Test thumbnail visibility in store search results
- A/B test different screenshot orders if possible
- Update screenshots with major UI changes

### Common Mistakes to Avoid
- Using outdated screenshots after UI updates
- Including personal information or test data
- Showing error states or loading screens
- Using low-resolution or scaled images
- Inconsistent visual style across screenshots
- Text overlays that obscure important UI
- Missing required device sizes
- Screenshots that don't represent actual app features
