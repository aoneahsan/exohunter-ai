# Feature Graphic / Banner Guide

## Overview

The feature graphic (Google Play) or promotional banner is a wide horizontal image that appears at the top of your app's store listing. It's one of the first visual elements users see and plays a crucial role in app store conversion.

## Google Play Store Requirements

### Feature Graphic (Required)
- **Dimensions:** 1024 x 500 pixels (exact)
- **Format:** PNG or JPG (24-bit, no alpha/transparency)
- **File size:** Maximum 1MB
- **File name:** `feature-graphic.png` or `feature-graphic.jpg`
- **Color space:** sRGB
- **Aspect ratio:** 2.048:1 (fixed)

### Display Locations
- App store listing header (web and mobile)
- Store search results (sometimes)
- Featured sections and promotions
- Google Play homepage carousel (if featured)
- App preview in recommendations

### Design Guidelines
- **Safe zone:** Keep important elements away from edges
- **Text readability:** High contrast, large enough to read on mobile
- **Brand consistency:** Match app's visual identity
- **Feature focus:** Highlight main value proposition
- **No device frames:** Show app UI or branded graphics
- **Scalability:** Looks good at various sizes (desktop and mobile)

## Apple App Store

### Promotional Artwork (Optional)
Apple doesn't require a feature graphic like Google Play, but you can provide:
- **App Store Promotional Artwork:** 1600 x 2400 pixels
- **Search Ads Banners:** Various sizes if using Apple Search Ads
- **Marketing materials:** For press kit and website

For App Store purposes, focus on screenshots and app preview videos.

## Content Guidelines

### What Works Well
- App branding (logo, name, tagline)
- Key visual elements from app (planets, stars, space theme)
- Main features or benefits illustrated
- Professional, polished design
- Eye-catching colors and composition
- Clear call to action or value proposition
- Consistent with app icon and screenshots

### What to Avoid
- Cluttered design with too many elements
- Small, unreadable text
- Low-quality or pixelated images
- Misleading imagery not in app
- Competitor references
- Prohibited content (violence, adult content, etc.)
- Copyright violations (unlicensed images, fonts)
- Device mockups that obscure content
- Excessive text that violates store policies

### Text Guidelines
- **Avoid superlatives:** "Best", "#1", "Top", "Leading" are prohibited
- **Be factual:** Describe actual features
- **Keep it short:** 3-7 words maximum for main headline
- **High contrast:** Text must be readable on all backgrounds
- **Professional fonts:** Clean, modern, sans-serif recommended
- **Hierarchy:** Main message should dominate visually

## Creating the Feature Graphic

### Method 1: Professional Design Tools (Recommended)

#### Adobe Photoshop
1. Create new document: 1024 x 500 pixels, 72 DPI, RGB
2. Add space background (stock photo or gradient)
3. Add planetary imagery, stars, effects
4. Add text layers with branding and headline
5. Apply effects, adjustments, filters
6. Export as PNG or JPG (quality 100%, sRGB)
7. Verify file size <1MB

#### Figma
1. Create frame: 1024 x 500 pixels
2. Design with components, images, text
3. Use plugins for effects and stock images
4. Export as PNG (@1x, no compression)
5. Optimize with TinyPNG or similar
6. Verify dimensions and file size

#### Canva (Easiest for Non-Designers)
1. Create custom size: 1024 x 500 pixels
2. Search templates for "Google Play Feature Graphic"
3. Customize with ExoHunter AI branding
4. Use space-themed backgrounds and elements
5. Add text with readable fonts
6. Download as PNG (highest quality)
7. Verify dimensions

## Technical Specifications

### File Requirements
- **Dimensions:** Exactly 1024 x 500 pixels (no variance allowed)
- **Format:** PNG (recommended) or JPG
- **Color mode:** RGB
- **Color profile:** sRGB (standard)
- **Bit depth:** 24-bit color
- **Transparency:** None (solid background required)
- **Compression:** Optimize to <1MB without quality loss

### Safe Zones
- **Top/Bottom margin:** 50 pixels (10% of height)
- **Left/Right margin:** 100 pixels (~10% of width)
- **Text safe zone:** Keep text within center 824 x 400 pixel area
- **Logo safe zone:** 50 pixels from all edges

### Quality Checklist
- [ ] Exact dimensions: 1024 x 500 pixels
- [ ] File size <1MB
- [ ] sRGB color space
- [ ] No transparency/alpha channel
- [ ] High resolution, no pixelation
- [ ] Text is readable at thumbnail size
- [ ] Important elements in safe zone
- [ ] Colors match app branding
- [ ] No prohibited content or claims
- [ ] Tested on light and dark backgrounds
- [ ] Looks good on mobile and desktop
- [ ] Professional, polished appearance

## Testing and Validation

### Preview Methods
1. **Play Console Preview:** Upload to draft and preview
2. **Desktop Preview:** View at actual size (1024x500)
3. **Mobile Preview:** Scale to phone width and view
4. **Thumbnail Preview:** Scale to 256x125 pixels
5. **Print Preview:** Print at actual size to check clarity

### A/B Testing (After Launch)
- Create 2-3 variants with different designs
- Test performance in Play Console experiments
- Monitor conversion rates and engagement
- Iterate based on data

### Common Issues
- **Text too small:** Increase font size, test at thumbnail size
- **Low contrast:** Adjust colors, add background overlays
- **Cluttered design:** Simplify, remove unnecessary elements
- **Pixelated images:** Use higher resolution source images
- **File too large:** Compress with TinyPNG or JPEGmini
- **Wrong dimensions:** Re-export at exact 1024x500 pixels

## Tips for Success

### Design Best Practices
- **Keep it simple:** One main focal point
- **High contrast:** Text must be instantly readable
- **Professional quality:** No amateur/clipart look
- **Brand consistency:** Match icon and screenshots
- **Test at scale:** View at various sizes before finalizing
- **Avoid trends:** Design should age well

### Common Mistakes
- Text too small to read on mobile
- Cluttered with too many elements
- Generic stock photo without branding
- Misleading content not in app
- Wrong dimensions causing distortion
- Using prohibited claims or superlatives
- Poor image quality or pixelation

### Optimization
- Use compelling imagery that represents app
- Make primary benefit obvious at a glance
- Include app name/logo for brand recognition
- Design for mobile viewing (primary use case)
- Test with non-users for clarity
- Update with major app redesigns
